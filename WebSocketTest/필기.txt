웹 소켓, WebSocket
- 소켓(Socket) > 네트워크상 호스트간의 데이터를 주고받는 규약(인터페이스)
- WebSocket > ws 프로토콜을 기반으로 웹 클라이언트(js)와 서버 사이에 통신을 제공하는 기술

개념 비교
1. 이전 환경
    - 웹서버 <- (데이터) -> 브라우저
    - 비연결 지향
    - 클라이언트의 의지만으로 서로 연결이 가능
    
2. 이전 환경(ajax)
    - 웹서버 <- (데이터) -> XMLHttpRequest
    - 비연결 지향
    - 클라이언트의 의지만으로 서로 연결이 가능
    
3. 소켓 환경(ws)
    - 웹 소켓 <- (데이터) -> 웹 소켓
    - 연결지향(***)
    - 전화기
    ex) 카톡
    
소켓 동작
1. 호스트A, 호스트B

2. 호스트A > 소켓(전화기) 생성 + IP주소(+포트번호)(전화번호) > 종단점(EndPoint)
3. 호스트B > 소켓(전화기) 생성 + IP주소(+포트번호)(전화번호) > 종단점(EndPoint)

4. 호스트B > 받는 사람 > 서버 역할 > 미리 수신 대기 준비
5. 호스트A > 거는 사람 > 클라이언트 역할 > 호스트 B 종단점 + 연결 시도
6. 호스트B > 연결 수락 > 연결 성립 > 호스트 A와 연결되어 있는 전용 소켓을 생성

7. 서로 연결 > 서로간의 데이터를 송수신

8. 호스트A, 호스트B > 소켓 종료(close) > 통신 종료
